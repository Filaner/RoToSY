Workflow
약품 배송 워크플로우
처방전 발행부터 환자 복약까지 7단계로 진행됩니다. 각 단계는 담당 역할이 명확히 분리되어 있습니다.
1
간호사 · /nurse
처방전 발행
환자 선택, 담당 의사, 우선순위(긴급/일반/정기), 약품 목록을 입력하고 처방전을 제출합니다.
POST /api/prescription
긴급 처방은 약사 대기열 최상단에 배치됩니다.
2
약사 · /pharmacist
OCR·Vision AI 검증 후 승인
대기열에서 처방전을 선택합니다. OCR 인식 결과와 Florence-2 Vision AI 신뢰도를 확인하고 약품명·수량 일치 여부를 검토합니다. 이상 시 수량 수동 수정 가능. 반려 시 사유를 입력하면 간호사 화면에 표시됩니다.
POST /api/prescription/{id}/approve POST /api/prescription/{id}/reject
3
로봇 암 (자동) · 관리자 감시
로봇 암 약품 집기
Doosan E0509가 ArUco 마커로 서랍 위치를 인식하고 약품을 집어 AMR에 적재합니다. 관리자는 카메라/비전 탭에서 실시간으로 감시합니다.
GET /api/vision/stream GET /api/vision/detections
4
약사 + 관리자 · 이중 확인
이중 적재 확인
약사와 관리자가 각각 독립적으로 "적재 확인" 버튼을 눌러야 합니다. 둘 중 하나라도 미확인이면 AMR 출발 버튼이 비활성화됩니다. 서버가 강제 검증하므로 UI 우회 불가입니다.
POST /api/system/mission/confirm {actor: "admin"|"pharmacist"}
5
관리자 → AMR
AMR 출발 및 배송
두 확인이 완료되면 관리자 화면의 AMR 출발 버튼이 활성화됩니다. 출발 후 AMR 상태(DELIVERING)가 모든 화면에 실시간 전파됩니다.
POST /api/amr/dispatch {destination: "3병동"}
도착 후 도어가 자동 개방되지 않으면 관리자가 강제 개방 명령을 보낼 수 있습니다.
6
간호사 · /nurse
약품 수령 확인
AMR이 도착하면 간호사 화면 최상단에 초록색 배너가 자동 표시됩니다. "약품 수령" 탭에서 도착 약품 목록을 확인하고 수령 버튼을 클릭합니다. 이상이 있으면 이상 신고를 입력합니다.
POST /api/system/mission/complete
7
간호사 → 환자
환자 복약
간호사가 환자에게 약을 전달합니다. 환자 화면의 배송 상태 바가 "도착 완료"로 업데이트되고, 간호사의 병동 투약 일정에서 완료 체크를 합니다.
미션 COMPLETED 관리자 KPI +1

Data Flow
실시간 데이터 흐름
모든 화면은 하나의 WebSocket에 연결하고, 서버가 10Hz로 전체 시스템 상태를 브로드캐스트합니다.
WebSocket 브로드캐스트 페이로드 구조 (10Hz)
{ "robot": { ← RoToSY :8000 HTTP 폴링 "robot_state_str": "STANDBY", ← STANDBY | MOVING | ERROR | ... "servo_on": true, "is_moving": false, "current_tcp": [x, y, z, rx, ry, rz], "current_joints_deg": [j1, j2, j3, j4, j5, j6] }, "robot_online": true, ← RoToSY 연결 여부 "amr": { ← ROS2 /amr/* 토픽 "status": "IDLE", ← IDLE|DELIVERING|ARRIVED|RETURNING|CHARGING|ERROR "battery": 85.0, ← 0~100 "pose": {"x": 0.0, "y": 0.0, "theta": 0.0}, "destination": "3병동" }, "door": { ← ROS2 /door/status 토픽 "status": "CLOSED" ← OPEN|CLOSED|ERROR }, "mission": { ← 서버 in-memory 미션 상태 "status": "IDLE", ← IDLE|AWAITING_CONFIRM|CONFIRMED|DISPATCHED|DELIVERING|ARRIVED|COMPLETED "pharmacist_confirmed": false, "admin_confirmed": false, "can_dispatch": false }, "nodes": { ← ROS2 노드 헬스 (3s/10s 기준) "arm_controller": {"status": "ONLINE", "last_seen": "14:30:15"}, "amr_controller": {"status": "UNKNOWN", "last_seen": null}, "vision_node": {"status": "UNKNOWN", "last_seen": null} }, "arduino": { ← ROS2 mock 또는 아두이노 HTTP push "temperature": 22.5, "humidity": 58.0, "status": "NORMAL" ← NORMAL|WARNING|OFFLINE } }

미션 상태 기계
IDLE → AWAITING_CONFIRM → CONFIRMED → DISPATCHED → DELIVERING → ARRIVED → COMPLETED
미션 상태파이프라인 단계환자 화면 단계AMR 출발 버튼

AWAITING_CONFIRM4 적재확인2 조제중비활성
CONFIRMED4 적재확인2 조제중활성 (양쪽 확인 시)

DELIVERING6 이동중3 배송중—
ARRIVED7 도착4 도착—

Setup Guide
빠른 시작
ROS2 없이도 데모 모드로 바로 실행됩니다. 실제 하드웨어 연동은 팀별 포인트를 참조하세요.
데모 모드 (ROS2 없음)
1
저장소 이동
cd /home/vboxuser/final_project/hospital_web
2
서버 실행
bash run.sh
# 또는 직접 실행
pip install -r backend/requirements.txt
python -m uvicorn backend.main:app \
  --host 0.0.0.0 --port 8080 --reload
3
브라우저 접속
localhost:8080/ — 관리자
localhost:8080/pharmacist — 약사
localhost:8080/nurse — 간호사
localhost:8080/patient — 환자
localhost:8080/docs — Swagger UI
실제 로봇 연동 순서
터미널 1 — RoToSY (포트 8000)
ros2 launch doosan_controller robot_controller.launch.py mode:=real
터미널 2 — 병원 웹 게이트웨이 (포트 8080)
ROS2 소싱 후 bash run.sh
RoToSY(:8000)가 없어도 웹 서버는 실행됩니다. 관리자 화면에 OFFLINE으로 표시됩니다.
팀별 연동 포인트
비전팀 routers/vision.py — MJPEG 스트림, detections POST
AMR팀 ros_bridge.py — /amr/* 토픽 이름 확인
로봇팀 drawer_metadata.py — ArUco ID·위치 재교정
전기팀 routers/plc.py 추가 — 컨베이어 연동
아두이노 POST /api/sensor/reading — 온습도 push
주의: hospital_web/ 디렉토리에서 실행해야 합니다. backend/ 안에서 실행하면 ModuleNotFoundError: No module named 'backend' 오류가 발생합니다.
